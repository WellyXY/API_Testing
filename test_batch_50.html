🎬 Snax API Batch Test

Generate videos in batch for up to 50 images

🚀 Features

  • Up to 50 images - Increased from 10 to 50
  • Smart batching - Configure tasks per batch
  • Adjustable delay - Avoid API limits between batches
  • Real-time progress - See batch and overall progress
  • Parallel processing - Run tasks in parallel per batch
  • Error handling - One task failing won't block others

📦 Batch Options

Batch size options:

  • 3 tasks per batch (recommended) - stable
  • 5 tasks per batch - balanced
  • 10 tasks per batch - faster
  • Process all at once - handle all tasks simultaneously

Delay between batches:

  • 2 seconds - fast
  • 5 seconds (recommended) - avoid rate limits
  • 10 seconds - conservative
  • 30 seconds - safest
⚠️ Notes:
  • Prefer smaller batches (3-5 tasks) to avoid limits
  • Be patient with large uploads; the queue is managed automatically
  • If errors occur, increase the delay between batches
  • Ensure sufficient API quota

🔧 How to use

  1. Start proxy server:
    python pika_proxy_server.py
  2. Open in browser:
    http://localhost:8000
  3. Configure API (Provider, Version, API Key)
  4. Select up to 50 images
  5. Set batch parameters (size, delay)
  6. Enter prompt (optional)
  7. Click Start
  8. Monitor progress
  9. Wait until all tasks complete
  10. Use One-click Download

📊 Example processing logic

Assume 20 images uploaded, batch size 3, 5-second delay:

Batch 1: images 1-3 (parallel) wait 5s Batch 2: images 4-6 (parallel) wait 5s Batch 3: images 7-9 (parallel) wait 5s Batch 4: images 10-12 (parallel) wait 5s Batch 5: images 13-15 (parallel) wait 5s Batch 6: images 16-18 (parallel) wait 5s Batch 7: images 19-20 (parallel)
